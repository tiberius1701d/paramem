You are filtering enriched personal facts from a voice assistant conversation. For each numbered fact, choose one action:
- IGNORE it — keep the fact. Default action; when uncertain, IGNORE.
- ADD its index to the drop list — only when one of the rules R1-R6 below matches.
Ground every decision against the transcript. Never echo, modify, or add facts.

Example — IGNORE: `{{"subject": "Alex", "predicate": "lives_in", "object": "Portland"}}` — supported by the transcript, so keep it.
Example — ADD: `{{"subject": "Person_1", "predicate": "has_name", "object": "Person_1"}}` — matches R1, so add its index to the drop list.

## Drop rules — ADD a fact's index when any rule matches

R1. Self-loop. `subject` and `object` are the same string, regardless of predicate
R2. Name-swap pair. Both `A has_name B` and `B has_name A` are present (also `named`, `is`, `equals`). Add both indices.
R3. Transcript contradiction. The transcript states a different value for the same subject and predicate.
R4. Conversation-role reference. Subject or object refers to a conversation-role label (e.g. "Assistant", "User", "Speaker", "the bot", "the model") rather than a real-world entity introduced by the transcript.
R5. Content-free object. Object provides no transcript-grounded claim — empty, whitespace-only, or a content-free placeholder (e.g. "Unknown", "None", "Various", "Something", "N/A").
R6. Namespaced system identifier. Subject or object is a system / automation identifier — a dot-separated or URI-shaped namespaced token (e.g. `media_player.sonos_office`, `sensor.temperature_kitchen`, `climate.kitchen_thermostat`) referring to a device, sensor, or service rather than a personal entity.

## Input

Each fact below is preceded by its zero-based index in square brackets. Use that index to refer to the fact in your output.

Conversation transcript:
{transcript}

Enriched facts (numbered):
{facts_json}

## Output

Return ONLY a single JSON object with the key "drop", whose value is an array of zero-based integer indices.

Example with two drops: {{"drop": [3, 5]}}
Example for a clean input where nothing matches a rule: {{"drop": []}}

Do NOT wrap the output in backticks, code fences, or any other prose.
Do NOT include the facts themselves. Do NOT modify any field of any fact. Do NOT emit the surviving (KEPT) indices — only the indices to drop.
